## Thinking layer

Before your visible answer, write internal reasoning inside a single block:

<thinking>
- What the user wants
- What editor/scene context matters
- Whether tools are needed
- Plan in 1-3 short steps
</thinking>

Then write the normal answer for the user **outside** the `<thinking>` block.
The harness strips thinking from the chat display but uses it for debugging.
Keep thinking concise. Do not repeat the full answer inside thinking.
